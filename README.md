# LLM Evaluations

Static benchmarks vs production evals, LLM-as-judge with its biases, the eval-framework landscape (Inspect AI, Braintrust, Phoenix, Langfuse), drift detection and red-teaming.

**Live index:** https://brendanjameslynskey.github.io/LLM_Hub_Evaluations/

## Presentations in this series

| # | Title | Status | Description |
|---|-------|--------|-------------|
| 01 | Eval Landscape | in development | Static benchmarks (MMLU, MMLU-Pro, GPQA, HumanEval, SWE-bench, MT-Bench) and what they miss; capability vs capability-in-context; the eval pyramid; cost structures; building your first eval set. |
| 02 | LLM-as-Judge | in development | Pairwise vs scoring; rubric, exemplars, CoT prompts; biases (positional, verbosity, self-preference, sycophancy); mitigation; constitutional eval; calibration; cost vs quality. |
| 03 | Eval Frameworks | in development | Inspect AI (UK AISI), Braintrust, Phoenix (Arize), Langfuse, OpenAI evals, deepeval &mdash; features, fit, decision matrix. |
| 04 | Production Evals | in development | Online eval, drift detection (input &amp; output distributions), golden sets, regression CI gating deploys, sampled human review, feedback loops, dashboards, cost models. |
| 05 | Red-Teaming | in development | Threat modelling; manual vs automated red teams; HarmBench, AdvBench, PAIR, JailbreakBench; defending and reporting; eval against jailbreaks; building an internal practice. |
